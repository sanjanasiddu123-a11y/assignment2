# рџ”„ Application Workflow

The authentication workflow of the application follows these steps:

1. The user opens the **Login Page**.
2. If the user does not have an account, they can navigate to the **Registration Page**.
3. If the user forgets their password, they can access the **Forgot Password Page**.
4. From there, the user proceeds to the **Reset Password Page**.
5. After successful login, the user is redirected to the **Dashboard Page**.

This workflow demonstrates how a typical authentication system operates in many web applications.
